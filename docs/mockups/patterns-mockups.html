Circadian Signature — design notes

  • 24 hours × 7 days: each cell is one hour on one weekday. Bright columns = consistent peak hours; bright rows = peak days; bright clusters = specific day/time habits.
  • Hours on x-axis (12a → 11p), days on y-axis (Mon top): horizontal orientation makes the card wide and short — fits dashboard flow.
  • Only 4 hour labels: 12a / 6a / 12p / 6p, with tick marks. Every hour labeled would be noise; hover reveals the exact hour.
  • Empty cells stay faintly visible (rgba(warm, 0.08)), not blank — the gaps are part of the shape.
  • Cells scale up on hover for tooltip readability without cramping the grid.
  • One sentence below the grid names the densest cluster in plain language — no "peak" or "power zone" framing.
  • Upgrade over existing card: current "When you get stuff done" uses 4 time-blocks × 7 days (28 cells) on last-14-days data. This is 24 hours × 7 days (168 cells) on all-time data.

Task Marination — design notes

  • Two layers, shared x-axis: top = where your active tasks currently sit, bottom = where tasks historically live when they get finished. Contrast reveals drift.
  • Bucket widths are fixed proportions, not data-driven — bucket positions are stable so you can eyeball movement week over week.
  • Cell opacity encodes count within the bucket. Empty buckets still show faintly so they don't feel like holes.
  • Top layer uses warm (active); bottom uses sage/neutral (ghost) — makes the two reads distinct at a glance.
  • "Aging gracefully" is deliberate. "Overdue" would turn this into a shame meter. The bucket still shows the count — it just refuses to moralize.
  • Bucket labels live below the tracks, not inside them, so the cells stay clean geometric forms.
  • Empty state: hide the bottom layer until there are ≥5 completed tasks — the "usually finish" claim needs a baseline to be meaningful.
